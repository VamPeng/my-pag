# 01. 技术目标与边界

## 技术目标

V1 技术方案需要满足以下目标：

- 支撑单人长期使用的事项管理系统
- 支撑“Mac 开发、Ubuntu 笔记本部署、多端 Web 使用”的实际工作流
- 支撑桌面端优先、移动端可访问的 Web 产品形态
- 数据主存储放在用户自己的 Ubuntu 笔记本上
- 支撑从多台个人设备访问 Ubuntu 笔记本上的服务，但不直接公网裸露数据库
- 为未来迁移、扩展或开放源码保留演进空间

## 技术边界

V1 当前不追求以下技术目标：

- 多人协作架构
- 多租户系统设计
- 云原生部署体系
- 自动离线同步与冲突合并
- 完整登录、注册、找回密码流程

## 关键技术结论

- V1 采用 Web 应用架构，而不是原生桌面客户端优先
- V1 采用前后端分离，开发阶段在 Mac 上联调，部署阶段在 Ubuntu 笔记本上运行
- V1 采用本地数据库，而不是浏览器本地存储
- V1 采用受控远程访问方案，而不是数据库公网暴露
