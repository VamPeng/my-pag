# 08. 待确认技术事项

当前版本的主要技术待确认事项已基本收敛完成。

## 已确认结论

- V1 不额外增加应用层固定口令或设备白名单校验
- V1 不要求自动备份脚本首发上线
- 本地数据库文件放在用户主目录下的独立应用数据目录
- 开发阶段先使用本机进程运行，部署阶段再补 Docker Compose

## 已采纳建议

### 关于数据库文件目录

可选方案：

1. 放在仓库目录内
2. 放在用户主目录下的独立应用数据目录
3. 放在系统级数据目录

已采纳：方案 2。

推荐示例：

- `/home/<user>/.local/share/my-pag/`
- `/home/<user>/data/my-pag/`

推荐原因：

- 不和代码仓库混放
- 重建仓库或重装依赖时不容易误删数据
- 手动备份和迁移更清晰

当前不推荐：

- 直接放在 Git 仓库目录里
- 直接放在临时目录里

### 关于开发阶段是否容器化

说明：

- 容器化可以简单理解为：把服务运行环境打包到一个标准化运行盒子里，避免“我电脑能跑、另一台电脑跑不起来”

可选方式：

1. 开发阶段直接用本机进程跑前后端
2. 开发阶段就使用 Docker 容器运行后端

已采纳：先本机进程开发，部署阶段再补容器化。

推荐原因：

- V1 还在快速收敛阶段，先减少环境复杂度
- Spring Boot + SQLite 本地直接开发更快
- 等功能稳定后再补 Docker Compose，成本更低

后续若产生新的技术未定项，再继续补充到本章节。
