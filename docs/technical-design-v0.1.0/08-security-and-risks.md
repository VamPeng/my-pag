# 08. 安全与风险

## 主要风险

### 单机可用性风险

- 数据和服务都在一台笔记本上
- 设备关机、休眠、断网时，外部访问会失效
- 长时间持续运行会带来额外耗电、发热和风扇噪音
- 如果机器资源本身紧张，持续运行服务会和日常开发、浏览器、IDE 争用内存与 CPU

### 数据安全风险

- 若宿主机损坏而没有备份，数据可能直接丢失
- 若访问链路配置不当，可能导致服务暴露超出预期

### 运维稳定性风险

- 笔记本长期作为服务端运行，稳定性不如专用主机
- 本地网络变化可能影响远程访问体验

## 控制措施

- 使用受控私有访问网络，不直接暴露数据库
- 对应用服务只开放必要端口
- 建立数据库文件定期备份机制
- 使用容器化部署，降低环境漂移
- 开发阶段尽量控制后台服务数量，避免在笔记本上同时堆叠过多常驻进程
- 关闭自动休眠或为服务运行时单独配置电源策略
- 关注磁盘空间，避免 SQLite 文件和日志长期增长无人处理

## V1 接受的限制

- 不保证高可用
- 不保证多地域访问性能
- 不保证离线编辑后自动同步

## 后续演进方向

- 如访问稳定性不足，再迁移到常开主机或云主机
- 如账号体系需要正式上线，再补认证和授权层
- 如单机 SQLite 成为瓶颈，再迁移到 PostgreSQL
