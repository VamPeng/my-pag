# 05. 任务字段定义

V1 每条任务包含以下字段：

- 标题：必填
- 备注：可选
- 所属目录：可选，未选择时归入未分类
- 进度：必填
- 预期完成时间：可选
- 优先级：可选
- 创建时间：系统自动记录
- 更新时间：系统自动记录
- 完成时间：完成时自动记录

说明：

- 轻量事项与标准任务共用同一套字段模型
- 不同点不体现在“字段不同”，而体现在“允许部分字段为空”
- 例如轻量事项可只填写标题，后续再补充目录、进度、预期完成时间等信息

## 进度

当前结论：V1 将原先拆分的“状态”和“进度”合并为一个字段处理。

合并原因：

- 状态与进度在个人事项场景下语义高度重叠
- 分成两个字段会增加填写和维护成本
- 首版更适合用一个统一字段完成展示和筛选
- UI 上可以用更直观的进度样式承载，但数据层先保持单字段定义

当前建议的进度集合：

- 未开始
- 进行中
- 已完成
- 搁置

说明：

- V1 不使用 0-100% 的数值进度
- V1 优先采用阶段型进度
- 后续如果需要更细粒度推进表达，再评估是否增加数值进度或其他补充字段

## 优先级

当前建议的优先级集合：

- 低
- 中
- 高

## 时间字段

当前结论：V1 不增加开始时间，只保留一个时间字段。

该字段当前定义为：

- 预期完成时间：用于表达希望在什么时间前处理或完成该事项

说明：

- 该字段比“截止时间”更宽松，既适用于正式任务，也适用于轻量事项
- 如果后续产品逐步走向更强任务管理，再评估是否重新拆分为开始时间、截止时间等字段
