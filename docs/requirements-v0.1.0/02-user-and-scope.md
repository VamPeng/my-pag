# 02. 用户与边界

## 目标用户

- V1 的核心使用者是产品创建者本人
- 当前需求优先围绕个人使用场景展开
- 产品后续存在开放给他人使用，或开源给他人部署使用的可能性

说明：虽然当前主要服务单人场景，但需求和数据模型不应从根上写死为“永远只支持一个人”。

## 使用边界

- V1 不做完整的多人协作产品能力
- V1 不做访客、成员、共享目录、共享视图等共享功能
- V1 需要具备账号主体
- V1 暂不提供面向终端用户的登录、注册流程
- V1 的账号可以先通过手动预置或手动配置方式实现

## 演进约束

虽然共享能力当前不纳入 V1，但产品需要为后续扩展保留基础条件：

- 数据层应优先采用便于迁移和扩展的数据库化设计
- 数据模型应基于账号归属设计，而不是强绑定单用户本地结构
- 后续若增加共享、同步或托管能力，不应推翻当前核心数据结构

## 数据迁移与备份

- V1 不强制要求完整导入导出能力首发上线
- 但数据结构需要为后续备份、迁移、导出预留空间
- 后续若开放产品或源码给他人使用，迁移能力会成为重要能力之一
