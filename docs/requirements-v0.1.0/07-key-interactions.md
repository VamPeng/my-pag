# 07. 关键交互要求

## 快速记录

- 应支持快速新增任务
- 默认最少输入标题即可创建
- 未明确归类时自动进入未分类
- 快速创建入口默认放在主界面下方输入框中
- 用户输入标题后，应打开详情区继续补充字段
- 详情补充完成后，需要用户主动保存才算创建成功

## 列表浏览

- 应支持按当前导航节点展示任务
- 应支持任务进度、优先级、预期完成时间等基础信息摘要显示
- 应支持切换筛选条件
- 应支持在当前列表内切换是否查看已完成事项
- 已完成事项在列表中展示时，应以浅色弱化样式呈现，和未完成事项形成明确区分

## 编辑体验

- 点击任务后可直接进入可编辑状态
- 可修改标题、备注、目录、进度、预期完成时间、优先级
- 编辑内容变更后，需要用户主动保存才生效

## 完成体验

- 列表中应支持一键将事项标记为已完成
- 详情区中也应支持通过修改进度完成事项
- 两种入口应保持结果一致

## 删除体验

- 删除事项前需要二次确认
- 删除后事项应先进入回收站
- 应避免误触导致数据直接永久丢失

## 搜索体验

- V1 搜索仅搜索标题
- 搜索行为应尽量直接、低成本，不在首版引入复杂搜索语法

## 响应式

- V1 需要同时兼顾桌面端和移动端可用性
- 移动端至少保证可录入、可查看、可完成任务
