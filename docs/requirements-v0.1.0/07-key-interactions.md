# 07. 关键交互要求

## 快速记录

- 应支持快速新增任务
- 默认最少输入标题即可创建
- 未明确归类时自动进入收件箱
- 快速创建入口默认放在主界面下方输入框中
- 用户输入标题后，应直接进入创建任务流程

## 列表浏览

- 应支持按当前导航节点展示任务
- 应支持任务进度、优先级、预期完成时间等基础信息摘要显示
- 应支持切换筛选条件
- 应支持在当前列表内切换是否查看已完成事项

## 编辑体验

- 点击任务后可查看详情并编辑
- 可修改标题、备注、目录、进度、预期完成时间、优先级

## 搜索体验

- V1 搜索仅搜索标题
- 搜索行为应尽量直接、低成本，不在首版引入复杂搜索语法

## 响应式

- V1 需要同时兼顾桌面端和移动端可用性
- 移动端至少保证可录入、可查看、可完成任务
