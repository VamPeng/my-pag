# 11. 当前结论摘要

截至 v0.1.0，已确认结论如下：

- 产品仅供个人使用
- V1 以个人使用为主，但后续可能开放产品或源码给他人使用
- 第一版会比极简待办更完整，但不做协作和同步
- V1 在设计取舍上优先保证结构完整，同时允许轻量事项低门槛录入
- 使用树形目录承载工作、生活及项目层级
- 目录在 V1 中主要承担分类作用，不扩展复杂列表语义
- 轻量事项与标准任务共用同一套字段模型，允许部分字段为空
- 状态与进度合并为单一进度字段，采用阶段型表达
- 时间字段仅保留“预期完成时间”
- 优先级保留低、中、高三档
- V1 采用单页面产品结构，桌面端以三栏布局为主
- 快速创建入口放在主界面下方输入框
- 输入标题后打开详情区补充字段，并通过显式保存完成创建或更新
- 事项详情采用直接编辑模式，但变更必须手动保存
- 完成事项同时支持列表一键完成和详情区修改进度两种方式
- V1 明确采用个人电脑上的本地数据库作为主存储
- 数据库部署在用户自己的 Ubuntu 电脑上，并通过非公网直暴露方式从外部访问
- V1 需要账号主体，但当前不提供完整登录注册流程，先支持手动预置账号
- 不提供未保存草稿自动恢复能力
- 删除事项需要二次确认，删除后进入回收站，且支持恢复并长期保留
- 需要提供今天、近期、逾期等时间视图
- “近期”视图支持可配置时间范围
- 已完成事项不作为单独导航入口，可在当前列表中保留展示，并以浅色样式弱化呈现
- V1 不做基础数据导出能力
- 删除目录时，目录下事项需要支持两种处理方式：归类到未分类，或一起删除
- V1 暂不包含子任务和任务关系能力
- V1 明确不做提醒通知、附件上传和终端用户登录注册流程
- 搜索范围先限制为标题字段
- 共享能力当前不做，但数据层需要为迁移和扩展做准备
