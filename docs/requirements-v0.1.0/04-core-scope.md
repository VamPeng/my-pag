# 04. 核心需求范围

## 任务管理

系统需要支持以下基础任务操作：

- 新增任务
- 编辑任务
- 完成任务 / 取消完成
- 删除任务
- 移动任务到其他目录
- 搜索任务
- 按状态、优先级、时间筛选任务

V1 范围说明：

- 删除任务在 V1 中的实际行为为移入回收站，而不是直接永久清除
- 暂不支持子任务
- 暂不支持任务之间的依赖、关联、前后置关系
- 以上能力保留为后续迭代方向，但不纳入当前版本核心范围

## 目录管理

系统需要支持树形目录管理，用于承载工作、生活及项目分类。

目录示例：

- 工作
- 工作 / A 项目
- 工作 / B 项目
- 生活
- 生活 / 家庭

目录能力要求：

- 支持多级目录
- 支持新增、重命名、删除目录
- 支持展开与折叠
- 任务归属到某一个目录节点
- 删除目录时，需要支持两种处理方式：将目录下事项归类到未分类，或一起删除

说明：

- V1 中目录主要承担分类与组织作用
- 暂不扩展为带有复杂行为的“任务列表系统”或其他高级语义容器
- V1 从体验设计上优先保证 3 层内目录清晰可用，但数据结构不强制限制层级深度

## 智能视图

除树形目录外，系统还需要提供全局时间视图，作为独立入口长期固定存在。

V1 智能视图包含：

- 收件箱：尚未归类的任务默认进入这里
- 今天：截止日期为今天的任务
- 近期：即将到期的任务
- 逾期：已超过截止日期且未完成的任务

说明：

- 智能视图入口应放在左侧导航顶部，和树形目录并列，而不是隐藏到某个页面内
- “已完成”在 V1 中不作为单独导航入口存在，而是通过当前列表中的筛选条件查看
- “近期”视图的时间范围应支持按配置调整，而不是写死为固定天数

## 搜索范围

- V1 搜索仅覆盖标题字段
- 备注内容暂不纳入搜索范围
