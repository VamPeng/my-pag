# 03. V1 设计原则

- 结构完整优先：V1 优先保证事项模型、目录体系、筛选方式和核心流程完整可用，而不是只追求极简录入
- 低门槛记录：虽然整体设计偏完整，但轻量事项仍应支持仅填写标题后快速创建，其他字段按需补充
- 组织结构清晰：事项既能按项目归类，也能按时间查看
- 渐进式完善：V1 先把核心功能做出来，再根据实际使用反馈优化细节和体验
- 信息不过载：第一版做“够用且完整”，不堆叠高级能力

## 当前取舍结论

- 在“录入速度优先”和“结构完整优先”之间，当前明确选择后者
- 对于想法、待确认事项等轻量记录，接受先仅填标题，后续再补字段
- 产品迭代策略以“核心功能先做出来，后续再打磨”为主，而不是在第一版过度追求精细设计
