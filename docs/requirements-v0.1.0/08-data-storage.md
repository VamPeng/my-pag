# 08. 数据存储要求

## 存储策略

- V1 明确采用数据库存储
- V1 数据存放在个人电脑上的本地数据库中
- 当前不采用上云数据库方案
- 当前部署环境为用户自己的 Ubuntu 电脑
- 需要支持从外网对这台个人电脑上的数据进行更新访问
- 外部访问方式优先采用非公网直暴露方案，例如内网穿透、VPN 或其他受控通道
- 数据结构需要为后续迁移和扩展保留空间
- 事项数据应明确归属于账号主体

## 持久化要求

- 刷新页面后数据不能丢失
- 数据写入需要以显式保存结果为准
- 不要求支持“编辑中未保存草稿自动恢复”能力

## 删除数据处理

- 被删除的事项不应立即永久清除
- 删除后的事项应进入回收站
- 回收站中的数据需要支持恢复
- 当前默认长期保留，不做自动清理

## 说明

- 当前结论已排除 `localStorage`、`IndexedDB` 作为 V1 主存储方案
- 当前主数据源应是本地数据库，而不是浏览器本地缓存
- 若后续技术方案需要本地缓存，也只能作为辅助层，而不是主数据源
- 由于数据部署在个人电脑上，后续技术方案需要额外评估设备在线、网络连通性和访问安全性
- 在“个人电脑部署 + 外部访问”前提下，需要补充评估无人值守运行风险，包括进程异常退出后的自动恢复、主机重启后的服务自启动，以及不在家场景下的远程恢复能力
