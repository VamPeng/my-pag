# 06. 页面与入口

## 主界面结构

V1 主界面建议包含以下区域：

- 左侧导航区：智能视图 + 树形目录
- 主内容区：任务列表
- 详情/编辑区：展示并修改当前选中任务

当前结论：

- V1 先以单页面产品形态实现，不拆分为多个独立业务页面
- 桌面端主结构采用左侧导航、中央列表、右侧详情的三栏布局

## 页面入口逻辑

- 需要看时间维度任务时，从智能视图进入
- 需要看某个项目或生活分类时，从树形目录进入
- 需要快速录入但暂时不想分类时，直接进入收件箱

## 创建入口

- V1 需要提供一个固定可见的快速创建入口
- 当前建议将入口放在主界面下方
- 入口形态为输入框
- 用户在输入框中填写任务标题后，应打开详情区进入创建流程

说明：

- 该入口优先服务“先记下来”这一高频动作
- 用户在详情区补充目录、进度、预期完成时间等字段后，通过显式保存完成创建

## 移动端原则

- 当前阶段不展开详细移动端页面设计
- 但主结构需要保留向移动端降级的空间
- 若进入移动端适配阶段，可将桌面端三栏结构调整为分层查看模式，例如先看列表，再进入详情
